## Тема: Принцип підстановки Лісков (LSP)
### 1. Що таке LSP

Принцип підстановки Лісков (Liskov Substitution Principle, LSP), сформульований Barbara Liskov, стверджує:

Об'єкти підкласу повинні мати можливість заміняти об'єкти базового класу без порушення коректності роботи програми.
Іншими словами, якщо клас B наслідує клас A, то код, який працює з A, повинен коректно працювати і з B, не знаючи про підміну.

### Чому це порушення LSP?

Код, який працює з Bird, очікує, що кожен птах уміє літати:
Якщо передати Penguin, програма впаде з винятком.

## Висновки

### Порушення LSP виникає, коли:
- підклас змінює очікувану поведінку базового класу
- підклас звужує функціональність
- підклас генерує винятки там, де базовий клас гарантує роботу
### Основні способи дотримання LSP:
Використовувати інтерфейси замість жорсткого наслідування
Розділяти відповідальність (SRP)
Не перевизначати методи з іншою семантикою
Проєктувати абстракції через поведінку, а не через ієрархію
