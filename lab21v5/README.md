

#  Лабораторна робота №21 – OCP: гнучкі алгоритми розрахунку (Strategy/Factory)

### 1. Мета роботи

Засвоїти принцип відкритості/закритості (OCP).
Реалізувати гнучку систему розрахунку вартості абонементів у спортзал.
Використати патерни Strategy та Factory, щоб додавати нові абонементи без змін сервісу

### 2. Завдання

Створити консольний проєкт GymPassLab.

Реалізувати інтерфейс стратегій IGymPassStrategy.

Створити три стратегії абонементів:

- MorningPass – ранковий абонемент зі знижкою.

- DayPass – денний абонемент.

- FullPass – преміум абонемент.

Використати Factory (GymPassFactory) для створення стратегії за вибором користувача.

Створити сервіс GymService, який використовує тільки інтерфейс стратегій.

Демонстрація роботи в Program.cs:

Користувач вводить тип абонементу, кількість годин та додаткові послуги (сауна/басейн).

Система розраховує та виводить вартість абонементу.

### 3. Результати

- Розрахунок вартості:

- Базова ціна: 5 * 10 = 50

- Сауна: +5 → 55

- Знижка ранкової години: 55 * 0.8 = 44

### 4. Висновки
Патерн Strategy дозволяє легко додавати нові абонементи без зміни коду сервісу.

Сервіс GymService залежить тільки від інтерфейсу IGymPassStrategy, що повністю відповідає принципу OCP (Open/Closed Principle).

GymPassFactory централізує створення стратегій, що спрощує розширення системи та підтримку коду.
