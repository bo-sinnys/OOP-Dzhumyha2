Проєкт: Lab20 – Реалізація SRP
1. Загальна характеристика

Необхідно розробити консольний застосунок для обробки замовлень із демонстрацією принципу єдиної відповідальності (Single Responsibility Principle).

Система повинна підтримувати створення, перевірку, збереження та підтвердження замовлень.

2. Функціональні вимоги

Система повинна забезпечувати:

Створення об’єкта замовлення (Order), що містить:

    ідентифікатор

    ім’я клієнта

    суму замовлення

    статус

    дату створення

Перевірку коректності замовлення:

    сума повинна бути більшою за нуль

    Збереження замовлення у сховищі (імітація бази даних).

    Відправлення повідомлення про підтвердження замовлення (через консоль).

    Оновлення статусу замовлення залежно від результату перевірки.

3. Архітектурні вимоги

Для дотримання SRP система повинна бути розділена на окремі компоненти:

    Валідатор замовлення (IOrderValidator)

    Репозиторій замовлень (IOrderRepository)

    Сервіс надсилання повідомлень (IEmailService)

    Координуючий сервіс (OrderService)

    Кожен компонент повинен відповідати лише за одну логічну задачу.

    Залежності мають передаватися через конструктор (Dependency Injection).

4. Нефункціональні вимоги

Код повинен бути структурований по папках (Models, Interfaces, Services, Initial).

Проєкт повинен компілюватися без помилок.

Повинна бути реалізована демонстрація роботи з валідним та невалідним замовленням.

5. Очікуваний результат

Програма повинна:

    коректно обробляти валідне замовлення;

    скасовувати невалідне замовлення;

    виводити інформацію в консоль;

    демонструвати правильну декомпозицію відповідно до SRP.
