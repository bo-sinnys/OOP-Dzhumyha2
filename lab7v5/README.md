## Лабораторна робота №7

### Тема: Обробка IO/мережевих помилок та патерн Retry

Мета роботи

Навчитися обробляти типові помилки вводу/виводу та мережеві помилки за допомогою блоків try-catch-finally, а також реалізувати патерн Retry для підвищення відмовостійкості застосунків.
Теоретичні відомості

У процесі роботи програм часто виникають помилки, пов’язані з роботою з файлами або мережею. У мові програмування C# такі помилки обробляються за допомогою механізму винятків (Exceptions).
Винятки вводу/виводу

До помилок вводу/виводу належать ситуації, коли програма не може виконати операцію з файлом або потоком даних. Найпоширеніші винятки:

- IOException
- FileNotFoundException
- DirectoryNotFoundException

Під час роботи з мережевими ресурсами можуть виникати такі винятки:

- HttpRequestException
- SocketException
- Конструкція try-catch-finally

Для обробки винятків використовується конструкція:

try
{
    // код
}
catch(Exception ex)
{
    // обробка помилки
}
finally
{
    // виконується завжди
}
Патерн Retry

Retry — це шаблон проектування, який дозволяє повторно виконувати операцію, якщо вона завершилась помилкою. Це особливо корисно для тимчасових помилок мережі або файлової системи.

Експоненційна затримка

При повторних спробах використовується експоненційна затримка, коли час очікування між спробами збільшується:

delay = initialDelay * 2^attempt

Це зменшує навантаження на систему.

### Хід роботи

Було реалізовано клас FileProcessor, який імітує роботу з файлами.
Метод UpdateConfig() перші два рази генерує виняток IOException, після чого завершується успішно.

Було реалізовано клас NetworkClient, який імітує мережевий запит.
Метод SendConfigUpdate() також генерує помилку HttpRequestException перші два рази.

Було створено допоміжний клас RetryHelper, який реалізує механізм повторних спроб виконання операції.

Основні можливості класу:

- повторне виконання операції
- логування спроб
- експоненційна затримка
- перевірка винятків через shouldRetry
  
У методі Main() продемонстровано:
- використання RetryHelper
- обробку помилок
- успішне виконання операції після декількох спроб

### Висновок

У ході виконання лабораторної роботи було вивчено механізм обробки винятків у мові програмування C#. Було реалізовано обробку помилок вводу/виводу та мережевих помилок за допомогою конструкції try-catch. Також було реалізовано патерн Retry, який дозволяє повторно виконувати операції у випадку тимчасових помилок. Використання експоненційної затримки дозволяє ефективніше керувати повторними спробами та підвищує відмовостійкість програмних застосунків.
